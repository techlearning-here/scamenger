# Expected output format from cloud story-creation code

When story content is generated by **cloud code** (or any external process), it should produce one file per story so you can copy it into the frontend. The CSV column **`story_file`** gives the expected filename for each row.

---

## 1. File name

- Use the **`story_file`** value from the CSV for that story (e.g. `pump-and-dump-fake-guru-investment-group.json`).
- Convention: **`{slug}.json`**, where `slug` is the story’s kebab-case identifier (same as in `scam-stories.ts` and the story URL).

---

## 2. File format: JSON

Each story output file must be **valid JSON** with a single object whose keys match the frontend’s `StoryContent` type.

### Required keys (order does not matter)

| Key | Type | Description |
|-----|------|-------------|
| `characterIntro` | string | First-person intro (1–3 sentences). |
| `initialPlot` | string | How they encountered the scam. |
| `scamExperience` | string | Mechanics: what happened. |
| `victimExperience` | string | What they thought/felt at the time. |
| `climax` | string | Turning point—realization or point of no return. |
| `victimPain` | string | Emotional and practical impact (after climax). |
| `learningVictim` | string | What they learned; red flags they see now. |
| `learningForReaders` | array of strings | 2–4 actionable takeaways (bullets). |

### Example JSON (minimal)

```json
{
  "characterIntro": "I was in my 30s, working full-time but worried about money.",
  "initialPlot": "I found a trading guru on social media with a big following.",
  "scamExperience": "The group pointed us to one platform. When I wanted to withdraw, they asked for fees.",
  "victimExperience": "I told myself I was being smart. Part of me wondered but I didn't want to miss out.",
  "climax": "One day they sold everything and the group was deleted. I found warnings online.",
  "victimPain": "I lost savings. Worse was the shame. It took months to report.",
  "learningVictim": "I now know: if someone directs you to one platform and tells you when to buy, it's a setup.",
  "learningForReaders": [
    "Watch for gurus and groups that push one platform and pressure to pay fees before withdrawal.",
    "Never invest through a link from a private group. Check with your bank or regulator.",
    "If you've lost money, report it to the FTC or your local authority."
  ]
}
```

### Formatting notes for integration

- **Paragraph breaks inside a section:** Use the Unicode paragraph character `\u2029` between paragraphs if you want to match the frontend’s stored strings (e.g. `"First paragraph.\u2029Second paragraph."`). Plain newlines can be normalized to `\u2029` during integration if needed.
- **Apostrophes in strings:** In the final TypeScript source, apostrophes inside single-quoted strings are written as `\u2019` where needed (e.g. `didn't` → `didn\u2019t`). The integration step or script can do this when writing each story into `frontend/src/data/scam-stories-content/{slug}.ts`.
- **Bold in text:** Use `**double asterisks**` around key phrases; the frontend renders them as bold.

---

## 3. Integrating cloud output into the frontend

After cloud code has written the story files (e.g. to a folder):

1. **Ensure listing exists:** The story must have an entry in `frontend/src/data/scam-stories.ts` in `SCAM_STORY_ENTRIES` with matching `slug`, `title`, and `category` (the CSV has this; the skill or your process can add missing entries).
2. **Add content:** For each `{slug}.json` file, create or update `frontend/src/data/scam-stories-content/{slug}.ts`:
   - The file should export a default `StoryContent` object (use the type from `./types`). Key = **slug** (filename without `.json`/`.ts`); value = the JSON object (string values and `learningForReaders` array), optionally converting newlines to `\u2029` and apostrophes to `\u2019` for TS strings.
   - After adding the new file, register it in `frontend/src/data/scam-stories-content/index.ts` (import and add to `STORY_CONTENT`).
3. **Update CSV:** Set that row’s `status` to `done` in the queue CSV so the next run skips it.

A small script can: read each `story_file` from the CSV, load the corresponding JSON, write `scam-stories-content/{slug}.ts`, and update `index.ts` to import and register the new story.

---

## 4. Reference: StoryContent type (frontend)

From `frontend/src/data/scam-stories-content/types.ts` (and re-exported from the folder):

```ts
export interface StoryContent {
  characterIntro: string;
  initialPlot: string;
  scamExperience: string;
  victimExperience: string;
  victimPain: string;
  climax: string;
  learningVictim: string;
  learningForReaders: string[];
}
```

Section order in the **writer doc** (for narrative flow) is: character intro → initial plot → scam experience → victim experience → **climax** → **victim pain** → learning (victim) → learning for readers. JSON keys can be in any order; the app uses the keys above.
